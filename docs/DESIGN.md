# Design

Last updated: 2026-04-25

Novelist should feel like a quiet writing desk: direct, fast, and visually
stable. The product should make long writing sessions easier without hiding
the fact that the document is plain Markdown.

## Interaction Principles

- First screen is a usable writing/project surface, not a marketing page.
- Writer controls should be discoverable through menus, command palette, and
  visible panels.
- Keep core writing controls close to the editor: file tree, tabs, outline,
  status, templates, and export.
- Avoid layout shifts in editor and tool surfaces.
- Keep CJK text readable and measurable in every mode.

## Visual Principles

- Prefer restrained desktop density over decorative layouts.
- Use icons and compact buttons for tool actions.
- Use clear text labels when the action is destructive, rare, or ambiguous.
- Avoid one-note palettes and oversized decorative sections inside the app.
- Preserve the user's content as the visual focus.

## Product Surface That Must Stay Visible

- Canvas, kanban, mindmap, templates, plugin panels, snapshots, stats, and
  export are core product features.
- Do not hide or demote them without explicit product approval.
