# File Renaming & Filename Macros (v0.2.4)

Status: Draft → shipping in v0.2.4
Last updated: 2026-05-07

This spec covers three small but related improvements to file naming UX
and consolidates the **filename / template macro convention** that was
previously split across `placeholder.ts` (filename slots) and
`template-tokens.ts` (snippet body variables).

## Goals

1. Make renaming a file or folder reachable without right-click.
2. Make the H1 → filename auto-rename feel reliably "automatic" — including
   when the user revisits a placeholder file in a later session.
3. Give users useful date / time macros in filename templates and snippet
   bodies, without breaking any existing tokens.

## Non-goals

- Live (per-keystroke) rename while typing the H1 — save-triggered is the
  contract, only the gating relaxes.
- Locale-aware date formats (`%x`, `%c`). Out of scope for v0.2.4.
- Renaming via inline-editable label inside `FileTreeNode`. The existing
  inline `<input>` row in `Sidebar.svelte` is reused unchanged.

## Feature 1 — Double-click renames

### Behavior

| Target          | Single-click                | Double-click |
|-----------------|-----------------------------|--------------|
| Folder row      | Toggle expand/collapse (NEW) | Enter rename mode (NEW) |
| Folder chevron  | Toggle expand/collapse (unchanged) | n/a |
| File row        | Open file (unchanged)       | Enter rename mode (NEW) |
| Disabled file   | No-op (unchanged)           | No-op |

Right-click → "Rename" still works. The rename UI itself is the existing
inline `<input>` row controlled by `Sidebar.startRename()`.

### Edge cases

- During an inline rename, the row renders as `<input>` instead of a
  `FileTreeNode`, so a double-click on the input field cannot loop back
  into another rename — it just selects text in the input.
- Drag-and-drop is unaffected: drag starts on `mousedown` + threshold,
  not `dblclick`.
- Folders that are also drop targets keep their `dragOver` highlight; the
  new single-click toggle does not interfere.

### Why folders change to single-click toggle

Without single-click toggle, double-click would either still expand (bad —
double-click means "rename") or do nothing on the row (worse — common Finder
muscle memory expects double-click to "open"). Migrating folder toggle to
single-click is the cleanest split: chevron stays as a redundant affordance.

## Feature 2 — H1 auto-rename: relax gating

### Current behavior

`tabs.svelte.tryRenameAfterSave` only renames if **all** of:

- `auto_rename_from_h1` setting is on
- `tab.justCreated === true` (file was created in this session)
- Filename matches `isPlaceholder()` (`Untitled N` / `第N章` / `Chapter N` / …)
- H1 is non-empty

### v0.2.4 behavior

Drop the `tab.justCreated` gate. Keep the other three gates.

Effect: if the user opens an existing placeholder file (e.g. they hit
Cmd+S a week ago without writing the H1, closed the app, came back), then
writes an H1 and saves, the file is renamed. Once renamed, the file no
longer matches `isPlaceholder()` and is permanently exempt — there is no
risk of a second auto-rename.

### Risk and mitigation

A user who manually creates `第1章.md` in Finder, fills in an H1, and saves
in Novelist **will** be auto-renamed. This is the original reason
`justCreated` existed. The trade-off is intentional: users have repeatedly
asked for "the rename to just happen", and `isPlaceholder()` is already a
narrow set. Users who want to keep raw chapter prefixes can disable
`auto_rename_from_h1` in Settings.

### Field cleanup

`TabState.justCreated` and the `OpenTabOptions.justCreated` parameter stay
in place but become no-ops. Callers are not changed (avoids a wide
refactor across IPC mocks and tests). A separate cleanup task removes
them after v0.2.5 if no consumers reappear.

## Feature 3 — Date / time macros and the unified naming convention

### The convention

Two distinct token shapes coexist; both use `{...}`:

| Shape                | Examples                  | Where it lives | Semantics |
|----------------------|---------------------------|---------|-----------|
| **Counter slot**     | `{N}` `{2N}` `{CN}` `{rN}` | filename templates only | Numeric placeholder filled by sibling-folder inference. UPPERCASE + optional leading qualifier. |
| **Variable**         | `{title}` `{filename}` `{project}` `{date}` `{time}` `{datetime}` | filename templates AND snippet bodies | Substituted at render time. Lowercase. Optionally takes a `:fmt` argument. |

This is a documentation change; existing tokens keep working.

### Validity rules for the filename template

A filename template is valid iff (after macro resolution):

- it contains **at most one** `{N}`-style counter slot;
- it contains **at most one** `{title}` slot;
- it contains **at least one** of the two;
- every remaining `{...}` brace-token is one of those two recognized
  shapes — so typos like `{cN}` (retired lowercase-Chinese token)
  or `{Title}` (wrong case) are rejected up front instead of
  silently becoming literal text in the filename.

`{title}` alone is valid: an empty H1 renders `Untitled.md` (then
`Untitled 2.md`, `Untitled 3.md` on collision), and once the user
writes an H1 the auto-rename pipeline replaces the whole stem.

### Date / time variables

```
{date}              → 2026-05-07
{date:YYYYMMDD}     → 20260507
{date:YYMMDD}       → 260507
{date:YYYY-MM-DD}   → 2026-05-07
{time}              → 14:08
{time:HHmm}         → 1408
{time:HHmmss}       → 140832
{datetime}          → 2026-05-07 14:08
{datetime:YYYYMMDD-HHmmss} → 20260507-140832
{datetime:timestamp}       → 1746626912    (Unix seconds, UTC)
{datetime:iso}             → 2026-05-07T14:08:32+08:00
```

Supported `fmt` tokens: `YYYY`, `YY`, `MM`, `M`, `DD`, `D`, `HH`, `mm`, `ss`.
Reserved keywords: `timestamp`, `iso`. Unknown tokens inside `fmt` are
emitted verbatim (e.g. `{date:WW}` → `WW`), so users can mix literal text:
`{date:YYYY年MM月DD日}` → `2026年05月07日`.

### Scope

- **Snippet body** (`resolveBody`): existing `{date}` / `{time}`
  no-format variants stay; new `:fmt` parsing added; new `{datetime}` added.
- **New-file filename template** (`services/new-file.ts`): the user's
  `template` string (e.g. `第{N}章-{date:YYMMDD}`) is now passed through
  `resolveBody(...)` *before* `parseTemplate(...)`. So a template like
  `{date:YYMMDD}-{N}.md` resolves to `260507-{N}.md` first, then numbering
  inference kicks in.
- **Template "new-file" filename** (`resolveFilename` in
  `template-tokens.ts`): already calls `resolveBody`, so it inherits the
  upgrade for free.

### Filename safety

Date formats can produce `:` (e.g. `{datetime:iso}`) or `T`. The existing
`sanitizeFilenameStem()` already strips disallowed characters before the
file is created, so this is not a new failure surface — but the docs
recommend using formats without `:` or `/` for filenames, and the in-app
hint surfaces a few safe presets.

### Macro hint UI

Next to the **Default filename template** input in Settings, a small
`?` icon (11×11, sidebar-icon-btn style). Hover or focus opens a popover
with:

1. Short cheat sheet table (counter slots + variables) using the same
   syntax as this spec.
2. Two example rows showing input → output:
   - `第{N}章-{date:YYMMDD}` → `第1章-260507.md`
   - `{datetime:YYYYMMDD-HHmmss}-{title}` → `20260507-140832-Untitled.md`
3. Link copy: "More: docs/product-specs/2026-05-07-v0.2.4-rename-and-macros.md".

i18n keys live under `settings.editor.newFile.macroHint.*` (en + zh-CN).

## Out of scope (revisit later)

- Per-folder template overrides (today the template is global + per-project).
- A dedicated date-picker / now-button in the template input.
- Localized month names (`MMM` / `MMMM`).
- Auto-rename on H1 *blur* without saving — would require coupling the
  rename pipeline to CodeMirror selection events.

## Acceptance

- Double-click on any sidebar file row enters rename mode; pressing Enter
  commits.
- Double-click on a folder row enters rename mode; single-click toggles
  expansion; chevron still toggles.
- After saving a placeholder file with an H1, the file is renamed even if
  the file was created in a previous session.
- `第{N}章-{date:YYMMDD}` is accepted as a template, previews three sample
  filenames in Settings, and produces a correctly-dated filename on Cmd+N.
- The `?` next to the template input shows a hover hint with the macro
  cheat sheet in both English and Simplified Chinese builds.
