# Publishing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Ship v0.2.4 publish feature: Ghost / WordPress (self-hosted + .com) / Medium with manual-paste auth, dialog-based metadata, no update tracking, pre-publish image upload to the publish target's own media endpoint.

**Architecture:** Per-platform Tauri commands backed by `core/src/services/publish/<platform>.rs`. Frontend orchestrator at `app/lib/services/publish.ts` routes by platform discriminant, runs pre-publish image rewrite, calls Pandoc for MD→HTML where needed. Mirrors the image-host architecture from the sister spec.

**Tech Stack:** Rust + Tauri v2 + reqwest + tokio + Svelte 5. Reuses existing Pandoc subprocess, `hmac`+`sha2` (Ghost JWT), wiremock dev dep.

## Phase 3 — Provider implementations

### Task 3.1: Medium (simplest — start here)

- `services/publish/medium.rs`.
- `pub async fn upload_image(token, bytes, filename, mime) -> Result<String, PublishError>` — multipart POST `/v1/images`, parse `data.url`.
- `pub async fn get_user_id(token) -> Result<String, PublishError>` — GET `/v1/me`, parse `data.id`.
- `pub async fn publish(config: &PlatformConfig, input: &PublishInput) -> Result<PublishResult, PublishError>`:
  - Resolve user id via `get_user_id`.
  - POST `/v1/users/{authorId}/posts` with body `{title, contentFormat: "markdown", content, tags, publishStatus, canonicalUrl}` (omit canonical if not set; default publishStatus = `public`).
  - Parse `data.url` and `data.id`.
- Tests: success, 401, 429, malformed JSON. Wiremock-stubbed.
- Commit `feat(publish): medium adapter`.

### Task 3.2: WordPress self-hosted

- `services/publish/wordpress.rs`.
- Helper `basic_auth_header(username, app_password) -> String` — `Basic <base64(user:pw)>`. Strip spaces from password.
- `pub async fn upload_image(site_url, basic_auth, bytes, filename, mime) -> Result<(String /* url */, u64 /* attachment_id */), PublishError>`:
  - POST `/wp-json/wp/v2/media` with body = raw image bytes, headers `Content-Type: <mime>`, `Content-Disposition: attachment; filename="<filename>"`, `Authorization: <basic_auth>`.
  - Parse `id` and `source_url`.
- `pub async fn resolve_tag_ids(site_url, basic_auth, tag_names: &[String]) -> Result<Vec<u64>, PublishError>`:
  - For each name: `GET /wp-json/wp/v2/tags?search=<name>`. If empty, `POST /wp-json/wp/v2/tags { name }`.
  - Collect ids.
- `pub async fn publish(config, input) -> Result<PublishResult, PublishError>`:
  - Resolve tag ids.
  - POST `/wp-json/wp/v2/posts` with body `{title, content (HTML), excerpt, slug, status, tags: [ids], featured_media: input.featured_media_id }`.
  - Parse `id`, `link`.
- Tests: success, 401, 5xx, tag-resolution-create, malformed response.
- Commit `feat(publish): wordpress self-hosted adapter`.

### Task 3.3: WordPress.com

- `services/publish/wordpress_com.rs`.
- Same wire shape as wordpress.rs except:
  - Base URL: `https://public-api.wordpress.com/wp/v2/sites/<site_id_or_domain>`
  - Auth header: `Authorization: Bearer <access_token>`
- Reuse helpers where reasonable but keep the file separate per architecture decision.
- Tests: same shape.
- Commit `feat(publish): wordpress.com adapter`.

### Task 3.4: Ghost (most complex — JWT signing)

- `services/publish/ghost.rs`.
- Helper `make_jwt(api_key) -> Result<String, PublishError>`:
  - Split `api_key` on `:` into `id` and `secret_hex`.
  - hex-decode `secret_hex` to bytes.
  - Header `{"alg":"HS256","kid":"<id>","typ":"JWT"}`.
  - Payload `{"iat":<now_secs>,"exp":<now_secs+5*60>,"aud":"/admin/"}`.
  - base64url(header) + "." + base64url(payload) + "." + base64url(HMAC-SHA256(secret_bytes, header.payload)).
- Helper `auth_headers(jwt) -> [(name, value)]` returning `Authorization: Ghost <jwt>`, `Accept-Version: v5.0`, `Content-Type: application/json`.
- `pub async fn upload_image(admin_url, api_key, bytes, filename, mime) -> Result<String, PublishError>`:
  - POST `/ghost/api/admin/images/upload/` (multipart, field `file`, optional `purpose=image`).
  - Parse `images[0].url`.
- `pub async fn publish(config, input) -> Result<PublishResult, PublishError>`:
  - Generate JWT.
  - POST `/ghost/api/admin/posts/?source=html` with body `{posts:[{title, html (input.body), tags: [{name}], status, slug, custom_excerpt, feature_image}]}`.
  - Parse `posts[0].id`, `posts[0].url`.
- Tests:
  - JWT-known-vector test: fixed `api_key`, fixed `now`, asserts the resulting token decodes correctly and the HMAC matches a precomputed signature.
  - Success path with wiremock.
  - 401, 5xx, malformed.
- Commit `feat(publish): ghost adapter with hs256 jwt`.
