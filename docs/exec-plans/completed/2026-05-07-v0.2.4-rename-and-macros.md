### F3a — date-format util (≈30 min)

1. Create `app/lib/utils/date-format.ts`:
   - `export function formatDate(d: Date, fmt: string): string`
   - Reserved fmt keywords short-circuit: `'timestamp'` → `Math.floor(d.getTime()/1000).toString()`; `'iso'` → ISO string with local offset (`+08:00` style).
   - Token regex: `/(YYYY|YY|MM|M|DD|D|HH|mm|ss)/g`, replacement function uses padded values from `d`.
   - Unknown chars in fmt are passed through (so `2026年MM月` works).
   - Year is `getFullYear()`, NOT UTC.
2. `tests/unit/utils/date-format.test.ts`: cover each token, padding, midnight, leap-year Feb 29, December 31 23:59:59, timestamp non-fractional, iso offset format.

### F3b — extend resolveBody (≈20 min)

1. `template-tokens.ts`: add `datetime: string` field to `TemplateContext`. Compute it as `${date} ${time}` in `makeTemplateContext`. Pass `now` through to all three.
2. Replace `resolveBody` regex with `/\{([a-zA-Z][a-zA-Z0-9_-]*)(?::([^}]*))?\}/g`. When `name` is `date|time|datetime` AND `fmt` is present, call `formatDate(now, fmt)` — but `resolveBody` doesn't currently take `now`. Add an optional `now?: Date` to `TemplateContext` (kept on the object) so the formatter can re-derive — OR pass the raw `Date` along separately. Cleaner: `TemplateContext` gets `_now: Date` (private-ish). For unknown variable names with no fmt, return the original `{name}` (matches current behavior). For known variables WITH fmt, hand off to formatter; for known variables WITHOUT fmt, return the precomputed string.
3. Tests: extend `tests/unit/utils/template-tokens.test.ts` (create if absent). Cover: known variable + fmt, known + no fmt, unknown + fmt (pass through), unknown + no fmt (pass through), `{datetime:timestamp}` and `{datetime:iso}`.

## Risk register

- **Folder single-click toggle muscle-memory regression:** users who learned double-click toggle (matched against `2025-09-08` baseline) will now trigger rename. Mitigation: rename UI is reversible (Esc cancels, no commit on blur with empty value). Acceptable.
- **`resolveBody` regex change:** the new optional `:fmt` group captures more, but unknown tokens with fmt should still be returned verbatim (`{nonsense:abc}` → `{nonsense:abc}`). Test explicitly.
- **Filename with `:`:** if a user writes `{datetime:iso}` in the template, `sanitizeFilenameStem` strips the colons and `T`. Document this in the macro hint ("dates with `:` will be sanitized in filenames").
- **Auto-rename now fires on saved-not-this-session placeholders:** acknowledged risk, see spec § Feature 2.
