# v0.2.6 Release Notes

**Date:** 2026-05-23
**Tag:** `v0.2.6`
**Package version:** `0.2.6`

## Theme

v0.2.6 was a reliability and release-hardening band. The visible release fixed
dirty-save safety, empty-file saves, H1 filename sync, DPI/zoom behavior,
sidebar readability, startup event handling, and Node 26 test stability. The
same post-v0.2.4 band also carried the Windows portable zip design and
single-file open routing work that shaped the next distribution and windowing
surface.

## Shipped

- Save and close flows now stop when a dirty file fails to save, avoiding silent
  data loss during tab close, window close, or project switch.
- Empty document saves write empty content instead of bypassing the save path.
- H1-driven filename sync works for `Untitled {N}`-style templates and respects
  the `auto_rename_from_h1` setting.
- First-launch and legacy auto-zoom behavior no longer compounds OS DPI scale.
- Sidebar file rows are more readable for CJK and non-monospace desktop
  navigation.
- Startup and menu event listeners log async failures without breaking the app
  shell.
- Vitest setup provides deterministic `localStorage` behavior under Node 26.
- Rust async tests that mutate process-global data directories were hardened.

## Verification Notes

The release-hardening surface for this repo is:

- `pnpm check`
- `pnpm test`
- `pnpm test:e2e:browser`
- `pnpm build`
- `cargo fmt --check`
- `cargo clippy --all-targets -- -D warnings`
- `cargo test`

For browser-visible regressions, `pnpm test:e2e:browser` is the preferred
truth signal when unit tests are polluted by local Node/Vitest storage setup.
