# v0.2.4 Release Notes

**Date:** 2026-05-14
**Tag:** `v0.2.4`
**Package version:** `0.2.4`

## Theme

v0.2.4 was the image hosting and publishing release. It also became a broad
writing-workflow polish release: filename macros, H1 rename behavior, CJK-aware
sorting/search fixes, sidebar interactions, settings search, security
hardening, and macOS localization all landed in the same release band.

## Shipped

- Image hosting for Qiniu Kodo, Aliyun OSS, S3/R2-compatible storage, imgur,
  sm.ms, and custom HTTP endpoints.
- Publishing adapters for Ghost, WordPress self-hosted, WordPress.com, and
  Medium.
- Settings panels for image hosts and publish channels, plus test actions for
  configured services.
- Editor share menu, publish dialog, image context-menu upload, and upload-all
  local images command.
- Filename templates with date/time macros and `{title}`-based H1 rename
  behavior.
- Sidebar and editor polish: double-click file rename, pane drop targets,
  edge-mounted sidebar toggle, wrap-long-file-names option, creation-time sort,
  Expand All / Collapse All, and better shortcut typography.
- Reliability and safety fixes for Pandoc discovery, Unicode search offsets,
  numeric CJK sidebar sorting, file watcher parent refresh, external links, zip
  template imports, and WebDAV path validation.

## Verification Notes

The release had broad Rust and frontend unit coverage, plus manual smoke tests
for the new image-host and publishing UI surfaces. The changelog explicitly
records that E2E coverage for those two new UIs was deferred after v0.2.4, so
future changes to either surface should add browser E2E coverage before another
large release.

## Follow-Ups

- Keep image hosting and publishing specs as shipped product contracts, not
  active plans.
- Add E2E coverage around the image-host and publish dialogs before expanding
  provider behavior.
- Continue moving durable decisions out of implementation plans and into
  `docs/design-docs/` when future work touches these surfaces.
